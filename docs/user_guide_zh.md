# 多方法时序因果发现比较分析平台 使用说明

## 1. 启动方式

在项目根目录打开终端后运行：

```powershell
python run_gui.py
```

也可以使用包入口：

```powershell
python -m app.main
```

## 2. 平台概述

本平台集成了两种平行的因果分析方法：

- **PCMCI+**：基于条件独立性检验的因果发现算法，适用于多元时间序列的因果结构学习。
- **TE（转移熵）**：基于信息论的因果分析方法，衡量变量之间的信息流动方向和强度。

两种方法相互独立、平行运行，各自输出结果。用户可根据需要选择同时运行两种方法，或只运行其中一种。

## 3. 界面流程

程序被组织为 4 步主流程。

### 第 1 步：数据导入

- 选择 Excel 或 CSV 文件。
- 点击"加载数据"。
- 左侧"列排除设置"用于标记不参与分析的列。
- 右侧"数据概览"会显示每一列的数据类型、是否为数值列、是否进入分析。

说明：

- 只有数值列会进入分析。
- 非数值列会自动忽略。
- 如果配置文件中提供了默认数据路径，启动后会自动填入路径框。

### 第 2 步：变量角色

这一步是当前版本最核心的设置入口。

可选角色：

- `普通变量`
  - 按常规 PCMCI 建模。
  - 既可以影响别人，也可以被别人影响。
- `时间驱动`
  - 不允许任何变量影响它。
  - 它可以影响普通变量，包括滞后影响和同时刻影响。
- `地形驱动`
  - 不允许任何变量影响它。
  - 它可以影响普通变量，包括滞后影响和同时刻影响。
- `排除变量`
  - 不进入本次分析。

推荐做法：

- 保持系统自动识别的 `date_sin`、`date_cos` 为 `时间驱动`。
- 保持系统自动识别的 `elevation_mean`、`slope_mean`、`aspect_mean` 为 `地形驱动`。
- 其他变量通常保持 `普通变量` 即可。

### 第 3 步：参数与运行

本步骤包含两个平行的参数设置标签页：

#### 3.1 PCMCI+ 因果发现

常用参数：

- `tau_min`：最小滞后期
- `tau_max`：最大滞后期
- `pc_alpha`：条件独立性检验的显著性水平
- `归一化分位数上下界`：数据归一化的分位数范围

#### 3.2 TE 转移熵分析

参数设置：

- `启用 TE 分析`：勾选后将在 PCMCI+ 分析完成后同时运行转移熵分析
- `分箱数`：数据离散化的分箱数量（默认 10），用于将连续数据转换为离散状态
- `历史长度 (k)`：转移熵的历史状态长度（默认 1），用于考虑目标变量的历史状态
- `最大滞后期`：TE 分析的最大滞后期（默认 3），分析变量之间的滞后信息流动
- `目标变量`：可选择特定变量作为分析目标，或选择"分析所有变量"

提示：TE 分析优先使用 `pyinform` 库；如果未安装，程序会自动改用内置离散 TE 估算。

#### 3.3 运行分析

点击"运行全部分析"后，程序会：

1. 先对数值数据做鲁棒缩放。
2. 根据变量角色和高级规则编译统一约束。
3. 调用 PCMCI+ 运行因果发现分析。
4. 如果启用了 TE 分析，继续运行转移熵分析。
5. 对结果矩阵做约束后处理，确保图和矩阵方向一致。
6. 自动保存 PCMCI+ 结果到 `outputs/pcmci_results.pkl`。

### 第 4 步：结果查看

结果页包含两个平行的结果展示标签页：

#### 4.1 PCMCI+ 结果

PCMCI+ 结果页包含：

- **因果图**：展示变量之间的因果关系网络
- **时间序列图**：展示时间序列变量之间的因果关系
- **目标变量上游影响图**：展示指向特定目标变量的上游因果链路
- **滞后邻接矩阵**：展示变量之间的滞后邻接关系
- **MCI 矩阵**：展示变量之间的 MCI 因果强度

其中"目标变量上游影响图"支持在结果页顶部切换目标变量。

- 如果存在 `10cm_mean`，程序会默认把它作为目标变量。
- 该图显示的是"所有最终会通向目标变量的完整上游链路"，不是只看直接影响。
- 同一方向如果在多个 lag 上都显著，只会折叠显示一条箭头，并在边标签中标注代表 lag 与 MCI。

当前"时间序列图"已经重构为论文版目标中心视图，并共享同一个目标变量下拉框。

- `仅显示指向目标变量的边`
  - 默认勾选。
  - 勾选后，只显示最终指向所选目标变量当前时刻 `(target, t)` 的边。
  - 取消勾选后，显示完整的时滞演化网络。
- `隐藏历史同期边`
  - 默认勾选。
  - 会隐藏 `t-1`、`t-2` 等历史列内部的同期连线，使画面更干净。
- `隐藏未决断因果边`
  - 默认勾选。
  - 会过滤 `o-o`、`x-x`、`o->` 等非严格定向边，只保留明确的 `-->` 因果箭头。

论文版时序图的视觉规则：

- 历史列节点为空心。
- 当前列节点为实心。
- 当前目标变量节点为小米橙实心。
- 当前其余变量节点为深灰蓝实心。
- 跨列影响以弧线箭头绘制，避免穿过节点。

#### 4.2 TE 结果

TE 结果页包含：

- **TE 矩阵**：展示变量之间的转移熵值，行表示源变量，列表示目标变量
- **NDTE 矩阵**：展示归一化方向性转移熵值（范围 0~1），值越大表示信息流动方向越明确
- **显著 TE 变量对**：列出所有 TE 值大于 0 的变量对，包括源变量、目标变量、滞后期、TE 值和 NDTE 值

TE 矩阵解读：

- TE 值衡量从源变量到目标变量的信息流动强度
- NDTE 值 = TE(X→Y) / (TE(X→Y) + TE(Y→X))，值越接近 1 表示信息主要从 X 流向 Y，越接近 0 表示主要从 Y 流向 X
- 值为 0 表示两个变量之间没有显著的信息流动

#### 4.3 导出结果

- 点击"导出 MCI 矩阵"可将 PCMCI+ 的 MCI 矩阵导出为 Excel 或 CSV
- 点击"导出 TE 结果"可将 TE 分析结果导出：
  - Excel 格式：包含 TE 矩阵、NDTE 矩阵和显著变量对三个工作表
  - CSV 格式：分别导出 TE 矩阵和 NDTE 矩阵两个文件

### 结果摘要

页面顶部的结果摘要区域会同时显示两种分析的摘要信息：

- PCMCI+ 分析结果：变量数、角色摘要、参数等
- TE 分析结果（如果运行了）：变量对总数、TE 贡献排名前 5

### 结果图主题

当前版本结果区统一采用白、灰、小米橙主题：

- 背景改为白色和浅灰层次，便于长时间查看。
- 结果图色条中点不再使用纯白，避免与背景混在一起。
- 因果图和时间序列图会按等比例绘制，节点不再被拉成椭圆。

## 4. 当前默认建模逻辑

本版本默认采用以下理解：

- 时间变量是外生驱动，只出不进。
- 地形变量也是外生驱动，只出不进。
- 时间和地形都允许对普通变量产生：
  - `tau>0` 的滞后影响
  - `tau=0` 的同时刻定向影响
- 地形不会再被强制做成"只保留自滞后"的静态节点。

这意味着：

- 其他变量不能影响时间。
- 其他变量不能影响地形。
- 时间可以影响其他变量。
- 地形也可以影响其他变量。

## 5. 常见问题

### 5.1 图里为什么没有某条边？

常见原因有两类：

- 统计上不显著，没有通过 `pc_alpha` 阈值。
- 该边被变量角色约束自动屏蔽。

### 5.2 为什么时间/地形变量看起来只有向外边，没有入边？

这是当前版本的预期行为，不是错误。

程序会把它们当作外生驱动变量：

- 不允许别人影响它们
- 允许它们影响普通变量

### 5.3 `tau=0` 同时刻边为什么容易出错？

因为 Tigramite 的同时刻关系在结果矩阵里是成对存储的。
如果只清一侧、不清另一侧，就会出现日志看起来删掉了，但图上仍有边的情况。

当前版本已经统一修复：

- 编译前约束
- 结果后处理
- 绘图前过滤

三层逻辑都使用同一套方向许可矩阵。

### 5.4 TE 分析失败怎么办？

常见原因：

- 未安装 `pyinform` 库：程序会自动改用内置离散 TE 估算；如果你希望使用 `pyinform` 后端，请运行 `pip install pyinform`
- 数据中存在过多 NaN 值：TE 分析要求数据完整，会在计算前自动删除含 NaN 的行
- 分箱数过大：建议使用默认值 10，或根据数据量调整

### 5.5 如何选择 PCMCI+ 和 TE？

两种方法各有优势：

- **PCMCI+**：适合发现因果图结构，输出明确的因果关系方向，支持条件独立性检验控制误报
- **TE**：适合量化信息流动强度，不需要假设因果图结构，对非线性关系也有一定敏感性

建议同时运行两种方法，综合对比结果。如果两种方法对同一对变量的因果方向判断一致，结果更可信。

## 6. 结果文件

默认保存位置：

```text
outputs/pcmci_results.pkl
```

你也可以在结果页手动导出 MCI 矩阵和 TE 结果。

## 7. 依赖安装

如果环境缺少依赖，可以执行：

```powershell
pip install pyqt5 pandas numpy matplotlib tigramite scikit-learn openpyxl pyyaml pyinform
```

其中 `pyinform` 是 TE 分析的可选加速/参考后端。如果不安装，程序会使用内置离散 TE 估算，PCMCI+ 也可正常运行。
