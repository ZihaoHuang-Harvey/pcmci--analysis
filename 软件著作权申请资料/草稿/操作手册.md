# 多算法因果分析平台 操作手册

## 1 软件概述

多算法因果分析平台是一款面向科研人员和数据分析人员的桌面端时间序列因果分析工具。软件集成了 PCMCI+ 和转移熵（Transfer Entropy）两种主流因果发现方法，提供图形化操作界面，帮助用户完成从数据导入、变量角色配置、参数调优到因果图可视化和结果导出的全流程分析工作。

本软件适用于科学研究与数据分析、测绘、遥感、地理信息等领域，主要服务于从事时间序列因果关系研究的科研人员，以及生态学、水文学、气象学等需要分析多变量间因果关系的学者。

软件的核心分析能力包括：基于条件独立性检验的 PCMCI+ 因果结构学习、基于信息论的转移熵信息流动分析、变量角色约束与领域先验知识融入、多种因果图可视化展示，以及分析结果的批量导出。

## 2 运行环境

### 2.1 操作系统要求

本软件运行于 Windows 10 或 Windows 11 操作系统。

### 2.2 依赖软件环境

运行本软件需要预先安装以下软件环境：

- Python 3.8 或更高版本
- PyQt5 图形界面库
- numpy、pandas 数据处理库
- matplotlib 图形绘制库
- scikit-learn 机器学习库
- tigramite 时间序列因果发现库
- pyinform（可选，用于加速转移熵计算）

### 2.3 硬件配置建议

建议使用 Intel Core i5 或同等性能处理器，内存不低于 8GB。分析大规模数据集时，建议使用 16GB 以上内存以获得更好的运行效率。

## 3 软件启动

在项目根目录下打开命令行终端，输入以下命令启动软件：

```
python run_gui.py
```

也可以使用模块方式运行：

```
python -m app.main
```

启动后，软件会打开主窗口。主窗口按照用户操作顺序组织为四个步骤标签页：数据导入、变量角色、参数与运行、结果查看。窗口顶部菜单栏提供"工具"菜单，其中包含时间序列拟合工具等辅助功能的入口。窗口底部状态栏会显示分析进度和日志信息。

## 4 数据导入

### 4.1 功能说明

数据导入是多算法因果分析平台的第一步操作。用户需要将待分析的时间序列数据加载到软件中，软件会自动识别数值列与非数值列，只有数值列会进入后续的因果分析流程。

### 4.2 操作步骤

1. 在主窗口的"第 1 步"标签页中，点击文件路径输入框右侧的"浏览"按钮，选择 Excel（.xlsx）或 CSV（.csv）格式的数据文件。
2. 点击"加载数据"按钮，软件会读取文件内容并在界面下方显示数据概览。
3. 数据概览区域会列出每一列的列名、数据类型、是否为数值列、是否参与分析等信息。非数值列会被自动标记为"不参与"。
4. 左侧"列排除设置"区域允许用户手动勾选不需要参与分析的列。被排除的列不会出现在后续分析中。
5. 确认数据加载和列选择无误后，进入"第 2 步"标签页。

### 4.3 数据格式要求

输入数据应为表格格式，每一列代表一个变量，每一行代表一个时间点的观测值。数据中需包含时间标识列（如日期列）和若干数值型观测变量列。如果配置文件中设置了默认数据路径，软件启动后会自动填入该路径。

## 5 变量角色配置

### 5.1 功能说明

变量角色配置是本软件的核心设置环节。不同的变量角色决定了该变量在因果分析中允许接收和发出因果影响的方向。软件支持四种变量角色，系统会根据变量名称自动识别常见的驱动变量，用户也可以手动调整。

### 5.2 四种变量角色

- **普通变量**：按常规 PCMCI 方式建模，既可以接收其他变量的因果影响，也可以对其他变量产生因果影响。大多数分析变量应设为此角色。
- **时间驱动**：不允许任何变量对其产生因果影响（无入边），但可以影响普通变量，包括滞后影响和同时刻影响。适用于 date_sin、date_cos 等时间编码变量。
- **地形驱动**：与时间驱动类似，不允许接收因果影响，但可以影响普通变量。适用于 elevation_mean、slope_mean、aspect_mean 等地形特征变量。
- **排除变量**：不参与本次分析，也不会出现在结果图中。适用于用户明确不需要纳入分析的变量。

### 5.3 操作步骤

1. 进入"第 2 步"标签页后，软件会列出所有数值列及其当前角色分配。
2. 系统会自动将名称包含 date_sin、date_cos 的变量识别为"时间驱动"，将名称包含 elevation、slope、aspect 的变量识别为"地形驱动"，其余变量默认为"普通变量"。
3. 用户可通过下拉菜单手动修改每个变量的角色。点击变量名称旁的角色选择框，从四种角色中选择合适的类型。
4. 界面下方会显示每种角色的简要说明，帮助用户理解角色含义。
5. 确认角色分配后，进入"第 3 步"标签页进行参数设置。

### 5.4 角色对分析结果的影响

设置为"时间驱动"或"地形驱动"的变量在因果图中只会出现向外发出的箭头，不会出现指向自身的箭头。这是因为约束引擎会在分析前将这些变量的入边方向屏蔽，确保分析结果符合领域先验知识。

## 6 PCMCI+ 因果分析

### 6.1 方法简介

PCMCI+ 是一种专门针对时间序列数据的因果发现算法。该方法通过条件独立性检验识别变量间的因果关系，能够同时处理滞后因果关系（某个变量过去时刻对当前时刻的影响）和同时刻因果关系。本软件使用偏相关检验作为条件独立性检验方法。

### 6.2 参数说明

在"第 3 步"标签页的"PCMCI+ 因果发现"子标签中，用户可以设置以下参数：

- **tau_min**（最小滞后期）：默认值为 0，表示考虑同时刻的因果关系。
- **tau_max**（最大滞后期）：默认值为 3，表示最多考察前 3 个时间步的滞后影响。增大此值可以发现更远距离的因果联系，但会增加计算时间。
- **pc_alpha**（显著性水平）：默认值为 0.05，用于判断因果关系统计显著性的阈值。值越小，筛选越严格，图中出现的边越少。
- **归一化分位数上下界**：默认为 25 和 75，用于数据鲁棒缩放的分位数范围。

### 6.3 运行操作

1. 在参数设置区域调整各参数值。
2. 勾选"启用 PCMCI+ 分析"（默认已勾选）。
3. 点击"运行全部分析"按钮，软件会在后台线程中执行分析，主界面不会冻结。
4. 进度信息会显示在窗口底部的日志区域，用户可以实时查看分析进展。
5. 分析完成后，软件自动跳转到"第 4 步"结果页面。

## 7 转移熵分析

### 7.1 方法简介

转移熵（Transfer Entropy）是一种基于信息论的因果分析方法，用于衡量从变量 X 到变量 Y 的信息流动方向和强度。与 PCMCI+ 不同，转移熵对非线性关系更加敏感。软件会计算 TE(X→Y) 和归一化方向性指标 NDTE，NDTE 接近 1 表示信息主要从 X 流向 Y，接近 0 则相反。

### 7.2 参数说明

在"第 3 步"标签页的"TE 转移熵分析"子标签中，用户可以设置以下参数：

- **启用 TE 分析**：勾选后将在 PCMCI+ 分析的同时运行转移熵分析。
- **分箱数（bins）**：默认值为 10，用于将连续数据离散化。数据量较大时可适当增加。
- **历史长度（k_history）**：默认值为 1，表示用于估算转移熵的历史步数。
- **最大滞后期（tau_max）**：默认值为 3，与 PCMCI+ 的 tau_max 含义相同。
- **分析全部变量对**：勾选后计算所有变量对之间的转移熵。

### 7.3 运行操作

转移熵分析可以与 PCMCI+ 同时运行，也可以单独运行。勾选"启用 TE 分析"后点击"运行全部分析"，软件会依次执行两种分析方法。如果未安装 pyinform 库，软件会自动使用内置的离散转移熵估算方法，功能不受影响。

## 8 高级约束设置

### 8.1 功能说明

高级约束设置面向已有明确领域先验知识的用户。通过设置边约束规则，用户可以精确控制哪些变量之间的因果关系允许或禁止出现在分析结果中。约束引擎以方向许可矩阵为核心，将变量角色约束和手动边约束统一编译后应用到分析过程中。

### 8.2 边约束类型

软件支持三种边约束类型：

- **no_out**（禁止出边）：指定变量不能对目标变量产生因果影响。
- **no_in**（禁止入边）：指定变量不能接收来自源变量的因果影响。
- **no_link**（禁止连接）：完全禁止源变量和目标变量之间的双向因果关系。

### 8.3 操作步骤

1. 在"第 3 步"标签页中找到"高级约束设置"区域。
2. 点击"添加约束"按钮，在弹出的对话框中选择源变量、目标变量和约束类型。
3. 可以添加多条约束规则，列表中会显示所有已添加的规则。
4. 对于不熟悉约束机制的用户，保持默认不设置通常更稳妥。变量角色配置已经覆盖了大多数常见的约束需求。

## 11 时间序列拟合工具

### 11.1 功能说明

时间序列拟合工具是多算法因果分析平台的辅助功能模块，用于基于不连续的时间序列数据补全缺失时间点的数值。该工具支持多种回归拟合方法，适用于遥感、生态等领域中因观测时间不连续导致的数据缺失问题。

### 11.2 启动方式

在主窗口菜单栏中点击"工具"→"时序拟合工具"，或在命令行中直接运行：

```
python app/tools/time_series_fitting_tool.py
```

### 11.3 操作步骤

1. 点击"加载数据"按钮，选择包含流域名称（watershed 列）、日期（date 列）和数值指标列的 Excel 或 CSV 文件。
2. 在"拟合方法"下拉菜单中选择回归模型，可选方法包括：Spline、CubicSpline、RandomForest、GradientBoosting、PLS、SVR、XGBoost、CatBoost。
3. 在"时间粒度"下拉菜单中选择输出数据的时间粒度：daily（日）、monthly（月）、quarterly（季）、yearly（年）。
4. 通过"平滑程度"滑动条调整拟合的平滑因子。
5. 点击"参数设置"按钮可以打开各模型的详细参数配置对话框。
6. 点击"全时序拟合"按钮开始执行拟合计算。
7. 拟合完成后，在结果标签页中查看各流域的拟合效果对比图。
8. 点击"导出结果"按钮将拟合后的数据导出为 Excel 文件。

## 12 常见问题与注意事项

### 12.1 因果图中为什么缺少某条边

可能的原因包括：该变量对之间的因果关系未通过 pc_alpha 显著性阈值（统计上不显著），或被变量角色约束屏蔽（例如目标变量被设为时间驱动，则不会有入边），或被高级边约束规则明确禁止。

### 12.2 TE 分析失败怎么办

首先检查是否安装了 pyinform 库，在命令行中运行 `pip install pyinform` 即可安装。如果已安装仍失败，检查数据中是否有过多的 NaN 值，可以尝试清理数据后重新分析。另外可以尝试减小分箱数（默认为 10），分箱数过大会导致每个 bin 内样本不足。

### 12.3 如何选择 PCMCI+ 和转移熵

PCMCI+ 适合发现因果图结构，能输出明确的因果方向和滞后阶数，适用于需要了解变量间因果网络拓扑的场景。转移熵适合量化信息流动强度，对非线性关系更敏感，适用于关注变量间信息传递方向和强度的场景。建议同时运行两种方法，综合对比分析结果，可以获得更全面的因果认识。

### 12.4 同时刻因果关系（tau=0）的注意事项

tau=0 的同时刻因果关系在实际数据中较难准确估计，容易受到混淆变量的影响。如果分析结果中同时刻边较多且不符合领域认知，可以尝试将 tau_min 设为 1，只考察滞后因果关系。
